**Pre-requisites (you must do before testing):**

1. Create PayFast sandbox accounts at https://sandbox.payfast.co.za
   - Register as a Buyer (for test payments)
   - Register as a Seller (for merchant credentials)
2. In your sandbox Seller account:
   - Go to Settings -> Integration -> Enable Subscriptions
   - Set a passphrase in Settings -> Integration -> Security
   - Note your Merchant ID and Merchant Key from Settings
3. Set Supabase Edge Function secrets:
   ```bash
   supabase secrets set PAYFAST_MERCHANT_ID=your_sandbox_merchant_id
   supabase secrets set PAYFAST_MERCHANT_KEY=your_sandbox_merchant_key
   supabase secrets set PAYFAST_PASSPHRASE=your_sandbox_passphrase
   supabase secrets set PAYFAST_SANDBOX=true
   supabase secrets set SITE_URL=https://your-frontend-url.com
   ```

**Test 1: Checkout redirect (PAY-02)**

PayFast sandbox does NOT support recurring billing. The recurring ITN handling (COMPLETE for recurring, FAILED, CANCELLED) is verified by unit tests only. It can only be fully tested in production with a real subscription. This is a known PayFast sandbox limitation documented in 04-RESEARCH.md.
