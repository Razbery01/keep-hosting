**A) `supabase/functions/create-payfast-order/index.ts`** — Server-side checkout param generator.

This function receives order details from OnboardingPage, generates the signed PayFast checkout parameters, and returns them to the client (which then auto-submits a hidden form to redirect to PayFast).

```
Import: jsr:@supabase/functions-js/edge-runtime.d.ts, cors.ts, supabase-admin.ts, payfast.ts
```

Handler flow:
1. Parse JSON body: `{ orderId, siteId, packageId, userEmail, userFirstName, userLastName }`
2. Validate all required fields present, return 400 if missing
3. Look up PACKAGES to get setup fee amount (use package price from body, or look up order from DB)
4. Read secrets: `PAYFAST_MERCHANT_ID`, `PAYFAST_MERCHANT_KEY`, `PAYFAST_PASSPHRASE`, `PAYFAST_SANDBOX` from `Deno.env`
5. Build params object per 04-RESEARCH.md Pattern 1:
   - `merchant_id`, `merchant_key`, `return_url`, `cancel_url`, `notify_url`
   - `name_first`, `name_last`, `email_address`
   - `m_payment_id` = orderId (our UUID)
   - `amount` = setup fee formatted as "999.00" (decimal, NOT cents)
   - `item_name` = `keep-hosting ${packageId} subscription`
   - `subscription_type` = `'1'`
   - `billing_date` = today YYYY-MM-DD
   - `recurring_amount` = `'49.00'`
   - `frequency` = `'3'` (monthly)
   - `cycles` = `'0'` (indefinite)
   - `custom_str1` = siteId
6. `return_url` = `${Deno.env.get('SITE_URL') || 'https://keep-hosting.co.za'}/dashboard`
7. `cancel_url` = `${Deno.env.get('SITE_URL') || 'https://keep-hosting.co.za'}/dashboard?payment=cancelled`
8. `notify_url` = `${Deno.env.get('SUPABASE_URL')}/functions/v1/payfast-itn`
9. Generate signature: `generatePayFastSignature(params, passphrase)`
10. Add signature to params
11. Determine PayFast URL: `getPayFastUrl(sandbox === 'true')`
12. Update order status to `'payment_pending'`
13. Return JSON: `{ params, payfast_url }` — client builds hidden form and submits

**B) `supabase/functions/payfast-itn/index.ts`** — ITN webhook handler. This is the critical path.

```
Import: jsr:@supabase/functions-js/edge-runtime.d.ts, cors.ts, supabase-admin.ts, payfast.ts
```

Handler flow:
1. Only accept POST. Return 405 for other methods.
2. Parse body as form-urlencoded: `const rawBody = await req.text(); const params: Record<string, string> = {}; new URLSearchParams(rawBody).forEach((v, k) => { params[k] = v })`
3. Extract key fields: `payment_status`, `m_payment_id` (our orderId), `pf_payment_id`, `custom_str1` (siteId), `token`, `amount_gross`
4. Verify signature: `verifyPayFastSignature(params, params.signature, passphrase)` — if false, log to `build_events` and return `jsonResponse({ error: 'Invalid signature' }, 200)` (MUST return 200 to prevent PayFast retries on permanently invalid requests)
5. **Idempotency check (PAY-04):** Query `orders` where `payment_id = m_payment_id`. If order exists AND status is already `'paid'` AND payment_status is COMPLETE, return `jsonResponse({ ok: true, skipped: 'duplicate' })` with 200. Log duplicate to `build_events`.
6. Log ITN receipt to `build_events`: `{ site_id: siteId, event_type: 'itn_received', status: 'info', message: JSON.stringify({ payment_status, m_payment_id, pf_payment_id }) }`

7. **COMPLETE handler:**
   - Look up order by `m_payment_id` (which equals our orderId): `supabase.from('orders').select('id, user_id, status, package').eq('id', m_payment_id).maybeSingle()`
   - If no order found with that ID, try looking up by `payment_id` field: `supabase.from('orders').select('...').eq('payment_id', m_payment_id).maybeSingle()`
   - Determine initial vs recurring: Check if subscription exists for this order. `const { data: existingSub } = await supabase.from('subscriptions').select('id').eq('order_id', order.id).maybeSingle()`.
   - **Initial COMPLETE (no existing subscription):** (PAY-05)
     - Update order: `status = 'paid'`, `payment_id = m_payment_id`
     - Insert subscription: `{ user_id: order.user_id, order_id: order.id, plan: order.package, status: 'active', payfast_subscription_id: pf_payment_id, payfast_token: token, next_charge_at: (today + 1 month).toISOString(), amount_cents: 4900, failed_charge_count: 0 }`
     - Fire-and-forget generate-site: `supabase.functions.invoke('generate-site', { body: { siteId } }).catch(() => {})`
     - Log to build_events: `itn_payment_complete_initial`
   - **Recurring COMPLETE (subscription exists):** (PAY-06)
     - Update subscription: `next_charge_at = (today + 1 month).toISOString()`, `failed_charge_count = 0`, `status = 'active'`
     - Log to build_events: `itn_payment_complete_recurring`
     - Do NOT trigger generate-site.

8. **FAILED handler:** (PAY-07)
   - Look up order by m_payment_id, get subscription
   - Update subscription: `status = 'grace_period'`, increment `failed_charge_count` (read current value, +1, write back)
   - Log to build_events: `itn_payment_failed`
   - Note: PayFast retries 3 times over 10 days. No server-side retry needed.

9. **CANCELLED handler:** (PAY-07)
   - Look up order by m_payment_id, get subscription
   - Update subscription: `status = 'cancelled'`, `suspended_at = now()`
   - Update order: `status = 'suspended'`
   - Fire-and-forget suspend-site: `supabase.functions.invoke('suspend-site', { body: { siteId } }).catch(() => {})`
   - Log to build_events: `itn_subscription_cancelled`

10. Always return 200 with `{ ok: true }` — even for unrecognized payment_status values, to prevent PayFast retries.

Wrap entire handler in try/catch. On error, log to console.error and build_events, return 200 (PayFast must not retry on our server errors).

7. Log to build_events
8. Return success — site stays live until PayFast sends CANCELLED ITN at end of billing period
