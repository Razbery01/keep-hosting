**B. `supabase/functions/deploy-site/index.ts`** — The core deploy function:

Imports: `jsr:@supabase/functions-js/edge-runtime.d.ts`, `corsResponse`/`jsonResponse` from `../_shared/cors.ts`, `getSupabaseAdmin` from `../_shared/supabase-admin.ts`, `JSZip` from `npm:jszip`.

Request body: `{ siteId: string }`

Flow (per 03-RESEARCH.md patterns — use them verbatim):

1. **Validate input** — return 400 if no siteId
2. **Log `deploy_start` event** to build_events
3. **Update build_status to `'deploying'`** on client_sites
4. **Rate limit check** (Pattern 6 from RESEARCH):
   - Query build_events for `deploy_done` events in last 60s — if >= 3, set `next_retry_at = now + 60s`, set `build_status = 'deploy_failed'`, log `deploy_rate_limited`, return early `{ queued: true, reason: 'rate_limit_per_min' }`
   - Query build_events for `deploy_done` events in last 24h — if >= 100, set `next_retry_at = tomorrow midnight UTC`, set `build_status = 'deploy_failed'`, return early
   - If >= 80 per day, log `deploy_quota_warning` event (but continue deploying)
5. **Read files from Storage** (Pattern 2 from RESEARCH):
   - `supabase.storage.from('client-sites').list(siteId, { limit: 100 })`
   - Filter: `fileList.filter(item => item.metadata !== null)` (skip folder entries)
   - Download each: `.download(\`${siteId}/${item.name}\`)` then `.text()`
   - Collect into `files: Array<{ path: string; content: string }>`
   - If no files found, set `build_status = 'deploy_failed'`, log error, return
6. **Create zip** (Pattern 3 from RESEARCH):
   - `const zip = new JSZip(); for (const f of files) zip.file(f.path, f.content);`
   - `const zipBuffer = await zip.generateAsync({ type: 'uint8array' })`
7. **Create Netlify site if needed** (Pattern 1 from RESEARCH):
   - Fetch current `netlify_site_id` from client_sites
   - If null: `POST /api/v1/${NETLIFY_TEAM_SLUG}/sites` with `{ name: \`kh-${siteId.slice(0, 8)}\` }`
   - Handle 422 (name taken) by appending 4 more chars: `kh-${siteId.slice(0,8)}${siteId.slice(8,12)}`
   - **CRITICAL: Write `netlify_site_id` and `netlify_url` (use `ssl_url`) to DB IMMEDIATELY** after site creation — BEFORE the zip deploy. This prevents orphan sites on retry (Pitfall 3).
   - If site already exists (netlify_site_id not null), skip creation
8. **Zip deploy** (Pattern 4 from RESEARCH):
   - `POST /api/v1/sites/${netlifySiteId}/deploys` with `Content-Type: application/zip`, body = zipBuffer
   - On 429: parse `Retry-After` header, set `next_retry_at`, set `build_status = 'deploy_failed'`, log, return
   - On other non-2xx: throw
   - Parse response for `deploy.id`
9. **Poll until ready** (Pattern 4):
   - Loop max 12 times, sleep 5s each: `GET /api/v1/deploys/${deploy.id}`
   - If `state === 'ready'`: break
   - If loop exhausts: set `build_status = 'deploy_failed'`, log timeout, return
10. **Mark deployed**:
    - Update `client_sites`: `build_status = 'deployed'`, reset `retry_count = 0`, `next_retry_at = null`
    - Log `deploy_done` event to build_events (this event is counted by rate limiter)
    - Return success

**Error handling:** Wrap entire flow in try/catch. On error: set `build_status = 'deploy_failed'`, log error event, return 500. Never leak secrets or internal paths in response.

const POLL_INTERVAL_MS = 5000
const MAX_POLL_ATTEMPTS = 12

**C. `supabase/functions/suspend-site/index.ts`** — Suspension function:

Request body: `{ siteId: string }`

Flow:
1. Validate siteId
2. Fetch `netlify_site_id` from client_sites — if null, return error (nothing to suspend)
3. Log `suspend_start` event
4. Create single-file zip: import `SUSPENDED_PAGE_HTML` from `../_shared/suspended-page.ts`, zip just `index.html`
5. Deploy placeholder zip to existing site: `POST /api/v1/sites/${netlifySiteId}/deploys`
6. On 429 or non-2xx: log error, return (do NOT change build_status — let caller retry)
7. Poll until ready (same pattern as deploy-site, but shorter timeout — 6 iterations max)
8. Update `client_sites.build_status = 'suspended'`
9. Update `orders.status = 'suspended'` where `order_id` matches (join through client_sites)
10. Log `site_suspended` event
11. Return success

**D. `supabase/functions/reactivate-site/index.ts`** — Reactivation function:

Request body: `{ siteId: string }`

Flow:
1. Validate siteId
2. Fetch `netlify_site_id` from client_sites — if null, return error (nothing to reactivate)
3. Log `reactivate_start` event
4. Update `build_status = 'deploying'`
5. Read files from Storage (same Pattern 2 as deploy-site — copy the same list+download+filter logic)
6. Create zip (same Pattern 3)
7. Deploy to existing site (same Pattern 4 — site already exists, no creation step)
8. On 429: set `build_status = 'deploy_failed'`, `next_retry_at`, return
9. Poll until ready
10. Update `client_sites.build_status = 'deployed'`, reset retry columns
11. Log `site_reactivated` event
12. Return success

NOTE: reactivate-site does NOT create a Netlify site (it already exists). It does NOT invoke generate-site or any Claude function. It reads directly from Storage and re-deploys.

For all three functions, follow the exact import pattern from existing Edge Functions: `import "jsr:@supabase/functions-js/edge-runtime.d.ts"` as first line, `Deno.serve(async (req) => { ... })` as the handler.
