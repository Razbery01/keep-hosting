**supabase/functions/deploy-site/index.ts** — The core deploy function. Follow the skeleton from 03-RESEARCH.md "Complete deploy-site Skeleton" section closely, with these specific requirements:

1. Import `jsr:@supabase/functions-js/edge-runtime.d.ts`, `npm:jszip`, `cors.ts`, `supabase-admin.ts`.
2. Define constants: `NETLIFY_API = 'https://api.netlify.com/api/v1'`.
3. Create `logEvent` helper (same pattern as persist-files).
4. On entry: parse `{ siteId }` from request body. Validate siteId exists.
5. **Rate limit check (DEPLOY-03):** Query `build_events` for `deploy_done` events in last 60 seconds (count). If >= 3, set `next_retry_at = now + 60s`, `build_status = 'deploy_failed'`, log `deploy_rate_limited` event, and return early with `{ queued: true, reason: 'rate_limit_per_min' }`. Then query last 24h count. If >= 100, set `next_retry_at` to start of next UTC day, `build_status = 'deploy_failed'`, return early. If >= 80, log `deploy_quota_warning` event (warning, do NOT block).
6. **Fetch site record:** Read `client_sites` row by `siteId`. If not found, return 404.
7. **Update status to `deploying`:** Set `build_status = 'deploying'`, `last_attempted_at = now()`. Log `deploy_start` event.
8. **Read files from Storage:** Use `supabase.storage.from('client-sites').list(siteId, { limit: 100 })`. Filter out folder entries with `.filter(f => f.metadata !== null)` (see 03-RESEARCH.md Pitfall 4). For each file, download via `.download(\`${siteId}/${item.name}\`)`, convert blob to text. If zero files found, throw error.
9. **Create zip:** `const zip = new JSZip(); files.forEach(f => zip.file(f.path, f.content)); const zipBuffer = await zip.generateAsync({ type: 'uint8array' })`. Always use `uint8array` type (03-RESEARCH.md Pitfall 5).
10. **Create Netlify site if needed (DEPLOY-02):** Only when `site.netlify_site_id` is null. Use `POST ${NETLIFY_API}/${teamSlug}/sites` (team slug endpoint, NOT personal `POST /api/v1/sites`). Body: `{ name: \`kh-${siteId.slice(0, 8)}\` }`. On success, **immediately write `netlify_site_id` and `netlify_url` to DB** (03-RESEARCH.md Pitfall 3 — write BEFORE deploy attempt). Use `ssl_url || url` from response for `netlify_url`.
11. **Deploy zip (DEPLOY-01):** `POST ${NETLIFY_API}/sites/${netlifySiteId}/deploys` with `Content-Type: application/zip`, body = zipBuffer. Handle 429: read `Retry-After` header (default 60), set `next_retry_at`, `build_status = 'deploy_failed'`, return early. On other non-ok responses, throw.
12. **Poll until ready (CRITICAL — 03-RESEARCH.md Pitfall 2):** Loop up to 12 iterations with 5-second sleep. GET `${NETLIFY_API}/sites/${netlifySiteId}/deploys/${deploy.id}`. Break when `status.state === 'ready'`. If loop exhausts, throw "Deploy timed out (60s)".
13. **Mark deployed:** Update `build_status = 'deployed'`, write `netlify_url`. Insert `deploy_done` event (this is the event that rate-limit counts). Reset `retry_count = 0`, `next_retry_at = null`.
14. **Error handling:** Catch block must set `build_status = 'deploy_failed'`, increment `retry_count`, log `deploy_error` event. Return 500 with error message. Re-read siteId from the request body (use a closure variable, not re-parsing).
