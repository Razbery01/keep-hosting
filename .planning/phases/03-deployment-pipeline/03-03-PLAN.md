### Step 1: Create Netlify Team Account + Set Secrets

1. Go to https://app.netlify.com and create a **dedicated Team account** for keep-hosting (or use existing one).
2. Note the **team slug** from the URL: `app.netlify.com/teams/{TEAM_SLUG}/...`
3. Go to **User Settings > Applications > Personal Access Tokens** and create a new token with deploy scope. Copy the token.
4. **CRITICAL: Check your pricing tier.**
   - Go to **Team Settings > Billing**.
   - If your account was created **after September 4, 2025**, you are on credit-based pricing: 300 credits/month, 15 credits per production deploy = only ~20 free deploys/month.
   - If your account is on a **legacy plan** (created before Sep 4, 2025), deploys are unlimited on the free tier.
   - If credit-based: consider upgrading to Pro ($19/mo, 1000 credits = ~66 deploys/month) before launch.
   - **Report your pricing tier** when you resume.

5. Set the secrets in Supabase:
   ```
   npx supabase secrets set NETLIFY_PAT="{your_token}"
   npx supabase secrets set NETLIFY_TEAM_SLUG="{your_team_slug}"
   ```

### Step 2: Remove GITHUB_PAT Secret (DEPLOY-01 locked decision)

Claude's Task 1 confirmed that no production code references GITHUB_PAT. It is safe to remove.

   ```
   npx supabase secrets list
   ```
   Confirm GITHUB_PAT is NOT in the output. NETLIFY_PAT and NETLIFY_TEAM_SLUG should be present.

### Step 3: Test Real Deploy

1. Find an existing `client_sites` row with `build_status = 'generated'` and files in Storage. (Or create one by running the generation pipeline on a test order.)
2. Invoke deploy-site directly:
   ```
   npx supabase functions invoke deploy-site --body '{"siteId": "{SITE_ID}"}'
   ```
3. Check the response -- should include `success: true` and a `netlify_url`.
4. **Visit the Netlify URL** -- the generated site should be live and accessible.
5. Check `client_sites` in the database -- `build_status` should be `deployed`, `netlify_site_id` and `netlify_url` populated.
6. Check `build_events` -- should have `deploy_start`, `deploy_done` events.

### Step 4: Test Suspend

1. Using the same siteId from Step 3:
   ```
   npx supabase functions invoke suspend-site --body '{"siteId": "{SITE_ID}"}'
   ```
2. **Visit the Netlify URL** -- should now show the branded "temporarily unavailable" placeholder page.
3. Check `client_sites` -- `build_status` should be `suspended`.
4. Check the linked `orders` row -- `status` should be `suspended`.

### Step 5: Test Reactivate

1. Using the same siteId:
   ```
   npx supabase functions invoke reactivate-site --body '{"siteId": "{SITE_ID}"}'
   ```
2. **Visit the Netlify URL** -- should now show the original generated site content (not the placeholder).
3. Check `client_sites` -- `build_status` should be `deployed`.
